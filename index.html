Detailed User Guide / How to Use

1. Preparation in Unreal Engine

Before using this tool, make sure you have generated the translation directories. In the Unreal Editor, open the Localization Dashboard, select your target (usually 'Game'), click Gather Text to capture texts from assets, and then add your target cultures. Finally, click Export Text if you want .po files, or just run the pipeline so that .archive files are generated under Content/Localization/Game/{CultureCode}/.

2. Scanning and Target Path Configuration

Copy the path of your Content/Localization directory (or a specific target directory like Content/Localization/Game) and paste it into the UE Project / Localization Path input field. Click Scan Folder. The tool will scan all subdirectories, detect the format (.archive or .po), map the corresponding languages, and show you a list of files ready for translation.

3. Adjusting Settings

Choose your Source Language (the language you wrote your game in, usually English). You can adjust the Delay (recommended at least 500ms for MyMemory API to avoid temporary IP blocks) and choose whether you want to overwrite existing translated texts or create backup copies of your files before modification.

4. Running Translation and Progress Tracking

Click Translate & Save All. The progress bar will appear at the top, showing which file is currently being processed and the overall completion percentage. The logs at the bottom will display real-time translation activities and any API errors.

5. Finalizing in Unreal Engine

Once translation is complete, go back to the Unreal Editor's Localization Dashboard. Click Import Text (if you translated .po files) and then click Compile Text. Compiling is mandatory for Unreal Engine to generate the binary .locres files, which are the actual files loaded by the engine during runtime to display your translated texts.
